# 数据准备专家交互设计说明

## 1. 设计目标

数据准备专家的交互目标不是替代现有数据源登记页面、审批流、防火墙开通流程或空间项目绑定流程，而是把这些既有流程串起来，让用户知道：

1. 从哪里开始登记数据源。
2. 当前流程走到哪一步。
3. 哪些步骤需要用户处理。
4. 哪些步骤由系统自动处理。
5. 数据源什么时候达到可使用状态。
6. 接入完成后可以继续做什么。

最终交互目标是将数据源推进到 `READY_FOR_USE` 状态，即完成数据源登记审批、元数据采集、数据地图更新和中台空间项目绑定，具备查询探索、离线开发、数据同步、安全扫描等后续使用条件。如果用户在登记时选择立即安全扫描，则需要在元数据采集后自动执行安全扫描，并完成安全等级确认总工单后再进入最终完成态。

## 2. 设计原则

1. 对话负责理解和反馈，流程负责确定性推进。
2. 不重做现有页面，通过统一工作台协议唤醒现有页面。
3. 不绕过现有审批流程，重点跟踪审批状态和提醒用户。
4. 元数据采集、数据地图更新等无审批步骤由系统自动执行。
5. 防火墙开通对接 IT 资源管理平台，只能异步发起和监听办结，不能同步等待。
6. 用户可在登记时选择是否立即采集表结构和执行安全扫描，选择后由系统自动推进。
7. 支持用户同时登记 PRD 和 UAT 数据源，并以一个任务展示两套数据源的准备状态。
8. 用户需要处理的节点必须给出明确入口、处理人、流程编号和下一步说明。
9. 所有长流程都必须支持状态查询、失败提示和恢复继续。

## 3. 整体交互框架

推荐采用“智能助手对话区 + 进度卡片 + 弹窗承载现有页面”的结构。

| 区域           | 作用                                 | 说明                         |
| -------------- | ------------------------------------ | ---------------------------- |
| 智能助手对话区 | 接收自然语言、解释进度、提醒下一步   | 用户主要入口                 |
| 进度卡片       | 嵌入在对话中展示数据准备全链路状态   | 解决用户不知道当前进度的问题 |
| 弹窗承载区     | 承载数据源登记表单、空间项目绑定表单 | 复用现有页面                 |

页面承载建议：

1. 现有页面最终采用弹窗承载。
2. 弹窗打开时携带 `sessionId/processId`，确保页面提交后可以回到数据准备流程。
3. 进度信息不放在独立右侧面板，优先作为对话卡片嵌入在智能助手消息中。
4. 现有页面无法在弹窗中承载时，可降级为新标签页，但必须保留 `sessionId/processId`。

## 4. 核心用户旅程

```text
用户自然语言发起
  -> 识别登记意图
  -> 唤醒数据源登记表单
  -> 用户填写并提交
  -> 直属领导审批
  -> 登记人复核
  -> 登记流程办结
  -> 系统自动连通性检测
  -> 如网络不通，异步发起防火墙开通流程
  -> 监听防火墙流程办结
  -> 系统自动采集元数据
  -> 系统自动更新数据地图、ES、Redis、元数据接口
  -> 如用户选择立即安全扫描，自动执行安全扫描
  -> 生成安全等级确认总工单
  -> 用户确认安全等级
  -> 唤醒或发送空间项目绑定表单
  -> 用户填写并提交
  -> 空间项目绑定流程办结
  -> READY_FOR_USE
  -> 推荐后续入口
```

## 5. 关键页面与状态

### 5.1 初始对话态

用户输入示例：

```text
帮我登记一个生产 MySQL 数据源
```

助手反馈：

```text
我来帮你准备数据源。已识别到环境为“生产”，数据源类型为“MySQL”。
我将打开数据源登记页面，并为你预填已识别的信息。
```

系统动作：

1. 创建数据准备任务 `processId`。
2. 提取可预填字段。
3. 唤醒现有数据源登记页面。
4. 初始化任务进度卡片。

### 5.2 数据源登记表单态

页面行为：

1. 打开现有数据源登记页面。
2. 预填环境、数据源类型等低敏字段。
3. 用户填写 IP、端口、database、schema、账号等字段。
4. 用户选择是否立即采集表结构。
5. 用户选择是否立即进行安全扫描。
6. 用户选择是否同时登记 PRD 和 UAT 数据源。
7. 用户提交表单。

助手侧反馈：

```text
请在右侧页面填写数据源信息。提交后会进入数据源登记审批流程，我会继续跟踪后续状态。

你也可以选择是否立即采集表结构和执行安全扫描；如果选择安全扫描，我会在元数据采集完成后自动发起扫描，并生成安全等级确认总工单。
```

提交成功后反馈：

```text
数据源登记流程已提交。

流程编号：PROC_xxx
当前节点：直属领导审批
我会持续跟踪审批状态，审批通过后自动进入元数据准备阶段。
```

### 5.3 审批等待态

进度卡片展示：

| 步骤               | 状态   |
| ------------------ | ------ |
| 数据源登记表单提交 | 已完成 |
| 直属领导审批       | 进行中 |
| 登记人复核         | 待开始 |
| 元数据采集         | 待开始 |
| 数据地图更新       | 待开始 |
| 绑定中台空间项目   | 待开始 |
| 接入准备完成       | 待开始 |

用户查询进度时：

```text
当前数据源登记流程正在等待直属领导审批。

流程编号：PROC_xxx
当前处理人：xxx
审批通过后会进入登记人复核，复核完成后我会自动触发元数据采集。
```

如果用户选择同时登记 PRD 和 UAT 数据源，进度卡片需要展示两个环境的子任务：

| 环境 | 当前节点 | 状态 |
| --- | --- | --- |
| PRD | 直属领导审批 | 进行中 |
| UAT | 登记人复核 | 待开始 |

助手反馈：

```text
你正在同时登记 PRD 和 UAT 数据源。我会分别跟踪两个环境的流程状态。

当前 PRD 数据源等待直属领导审批，UAT 数据源等待登记人复核。
任一环境出现阻塞时，我会单独提示。
```

### 5.4 登记人复核态

助手反馈：

```text
直属领导已审批通过，当前等待登记人复核。

复核完成后，我会继续执行连通性检测、元数据采集和数据地图更新。
```

异常反馈：

```text
数据源登记复核未通过。

原因：xxx
你可以根据复核意见修改后重新提交。
```

### 5.5 防火墙异步开通态

如果连通性检测失败且原因是网络不可达，数据准备专家需要异步发起防火墙开通流程。防火墙开通流程对接 IT 资源管理平台，不能同步等待结果，只能记录流程编号并监听办结消息或定时查询流程状态。

助手反馈：

```text
连通性检测未通过，原因是网络暂不可达。

我已为你发起防火墙开通流程：
流程编号：FW_xxx
当前状态：等待 IT 资源管理平台处理

该流程是异步处理，办结后我会自动重新检测连通性，并继续后续数据准备步骤。
```

进度卡片展示：

| 步骤 | 状态 |
| --- | --- |
| 数据源登记流程 | 已完成 |
| 连通性检测 | 失败 |
| 防火墙开通 | 进行中 |
| 元数据采集 | 待开始 |
| 数据地图更新 | 待开始 |
| 绑定中台空间项目 | 待开始 |
| 接入准备完成 | 待开始 |

防火墙流程办结后反馈：

```text
防火墙开通流程已办结，我正在重新检测数据源连通性。
```

### 5.6 自动准备态

登记流程办结后，系统自动进入无审批步骤：

1. 连通性检测。
2. 如果用户选择立即采集表结构，则执行元数据采集。
3. 元数据采集完成后，更新数据地图、ES、Redis、元数据对外接口。
4. 如果用户选择立即安全扫描，则执行安全扫描。
5. 安全扫描完成后生成安全等级确认总工单。
6. 如果用户未选择立即采集表结构，则进入“待采集表结构”状态，并提供后续继续采集入口；此时不自动更新数据地图，也不自动触发安全扫描。

助手反馈：

```text
数据源登记流程已办结，正在自动执行数据准备步骤。

当前步骤：元数据采集
任务编号：META_xxx
```

进度卡片展示：

| 步骤               | 状态   |
| ------------------ | ------ |
| 数据源登记表单提交 | 已完成 |
| 直属领导审批       | 已完成 |
| 登记人复核         | 已完成 |
| 连通性检测         | 已完成 |
| 元数据采集         | 进行中 |
| 数据地图更新       | 待开始 |
| 安全扫描           | 待开始 |
| 安全等级确认       | 待开始 |
| 绑定中台空间项目   | 待开始 |
| 接入准备完成       | 待开始 |

### 5.7 安全扫描与安全等级确认态

如果用户在登记时选择立即安全扫描，元数据采集完成后自动触发安全扫描。

助手反馈：

```text
表结构已采集完成，正在自动执行安全扫描。

扫描任务编号：SEC_SCAN_xxx
该步骤不需要你处理，扫描完成后会生成安全等级确认总工单。
```

安全扫描完成后：

```text
安全扫描已完成，我已为你生成安全等级确认总工单。

工单编号：SEC_LEVEL_xxx
处理入口：xxx

请确认安全等级。确认完成后，你可以继续进行数据脱敏。
```

用户完成安全等级确认后：

```text
安全等级确认已完成。

你现在可以继续进行数据脱敏。我也会继续推进中台空间项目绑定状态，直到数据源达到 READY_FOR_USE。
```

### 5.8 空间项目绑定态

元数据和数据地图准备完成后，助手反馈：

```text
数据源已完成登记、元数据采集和数据地图更新。

接下来需要绑定中台空间项目。我已为你打开绑定表单，并预填数据源信息。
```

如果通过待办发送：

```text
数据源已完成登记、元数据采集和数据地图更新。

接下来需要绑定中台空间项目，我已将绑定表单流程发送到你的待办。

流程编号：PROC_xxx
处理入口：xxx
完成绑定后，数据源将进入接入准备完成状态。
```

### 5.9 完成态

完成反馈：

```text
数据源 ds_xxx 已完成接入准备，并已绑定中台空间项目。

你现在可以继续：
1. 查询探索：查看表结构和样例数据
2. 离线开发：基于该数据源创建开发任务
3. 数据同步：配置同步链路
4. 安全扫描：扫描敏感字段和安全风险
5. 数据脱敏：基于已确认的安全等级配置脱敏规则
```

完成态需要展示：

- 数据源 ID
- 数据源名称
- 数据源类型
- 所属环境
- 数据地图入口
- 中台空间项目绑定状态
- 当前状态：`READY_FOR_USE`
- 后续可选入口

## 6. 进度卡片设计

### 6.1 基础信息区

展示字段：

| 字段         | 说明                           |
| ------------ | ------------------------------ |
| 数据源 ID    | 用户提交或系统生成的数据源标识 |
| 数据源类型   | MySQL、Oracle 等               |
| 所属环境     | 开发、测试、生产               |
| 环境登记模式 | 单环境、PRD/UAT 同时登记        |
| 当前状态     | 当前流程状态                   |
| 当前处理人   | 审批或待办处理人               |
| 流程编号     | 登记流程、防火墙流程或绑定流程编号 |
| 流程来源     | 数据源登记、IT 资源管理平台、空间项目绑定等 |
| 数据地图入口 | 准备完成后展示                 |

### 6.2 时间线状态

推荐状态枚举：

| 状态     | 说明                   |
| -------- | ---------------------- |
| 待开始   | 尚未进入该步骤         |
| 进行中   | 系统或用户正在处理     |
| 等待审批 | 等待审批人处理         |
| 等待用户 | 等待当前用户填写或提交 |
| 等待外部流程 | 等待 IT 资源管理平台等外部流程办结 |
| 已完成   | 步骤已完成             |
| 失败     | 步骤失败，需要处理     |
| 已跳过   | 当前场景不需要执行     |

### 6.3 操作按钮

根据当前状态动态展示：

| 当前状态             | 操作                                                 |
| -------------------- | ---------------------------------------------------- |
| 填写中               | 打开登记页面                                         |
| 直属领导审批中       | 查看审批流程                                         |
| 登记人复核中         | 查看流程详情                                         |
| 防火墙开通中         | 查看 IT 资源管理平台流程                             |
| 安全扫描中           | 查看安全扫描任务                                     |
| 等待安全等级确认     | 打开安全等级确认总工单                               |
| 元数据采集失败       | 重新采集                                             |
| 数据地图更新失败     | 重试更新                                             |
| 等待用户填写绑定表单 | 打开绑定表单                                         |
| 接入准备完成         | 查看数据地图、查询探索、离线开发、数据同步、安全扫描、数据脱敏 |

## 7. 对话话术设计

### 7.1 意图识别成功

```text
我来帮你准备数据源。已识别到数据源类型为“MySQL”，环境为“生产”。
接下来我会打开数据源登记页面，并跟踪后续审批、采集和空间项目绑定进度。
```

### 7.2 信息不足

```text
我可以帮你登记数据源。请先选择数据源类型和环境，我会打开对应的登记页面。
```

### 7.3 审批等待

```text
数据源登记流程已提交，当前等待直属领导审批。
审批通过后会进入登记人复核，复核完成后我会自动推进元数据采集。
```

### 7.4 自动步骤执行中

```text
数据源登记流程已办结，正在自动采集表结构。
该步骤不需要你处理，我会在完成后继续更新数据地图。
```

### 7.5 防火墙异步处理中

```text
连通性检测未通过，已发起防火墙开通流程。

流程编号：FW_xxx
流程平台：IT 资源管理平台
当前状态：处理中

该流程是异步处理，办结后我会自动重新检测连通性。
```

### 7.6 安全扫描执行中

```text
表结构已采集完成，正在自动执行安全扫描。
扫描完成后会生成安全等级确认总工单，请你确认后继续数据脱敏。
```

### 7.7 待采集表结构

```text
数据源登记流程已办结。你本次未选择立即采集表结构。

当前状态：待采集表结构
你可以稍后继续采集，采集完成后我会继续更新数据地图，并根据你的选择执行安全扫描。
```

### 7.8 等待安全等级确认

```text
安全扫描已完成，当前等待你确认安全等级总工单。

工单编号：SEC_LEVEL_xxx
处理入口：xxx

确认完成后，你可以继续进行数据脱敏。
```

### 7.9 等待用户绑定空间项目

```text
数据源基础准备已完成，当前还需要绑定中台空间项目。
我已为你打开绑定表单，请完成提交。绑定完成后，数据源将进入可使用状态。
```

### 7.10 完成

```text
数据源已完成接入准备，当前状态为 READY_FOR_USE。
你可以继续进行查询探索、离线开发、数据同步、安全扫描或数据脱敏。
```

## 8. 异常交互

| 异常场景           | 用户反馈                                       | 操作                       |
| ------------------ | ---------------------------------------------- | -------------------------- |
| 页面唤醒失败       | 登记页面打开失败，请重试或进入原系统处理       | 重试打开、复制入口         |
| 预填失败           | 已打开页面，部分字段需要手工填写               | 继续填写                   |
| 审批驳回           | 数据源登记审批被驳回，请根据意见修改后重新提交 | 查看驳回原因、重新打开表单 |
| 复核不通过         | 登记人复核未通过，请根据意见补充信息           | 查看复核意见、重新提交     |
| 防火墙流程发起失败 | 防火墙开通流程发起失败，请稍后重试或联系运维   | 重新发起、查看错误详情     |
| 防火墙流程处理中   | 防火墙开通正在 IT 资源管理平台处理中           | 查看流程、等待办结         |
| 防火墙流程驳回     | 防火墙开通流程被驳回，请根据意见处理           | 查看驳回原因、重新发起     |
| 元数据采集失败     | 数据源已登记，但元数据采集失败                 | 重新采集、联系管理员       |
| 数据地图更新失败   | 元数据已采集，数据地图更新稍后重试             | 重试更新、后台补偿         |
| 安全扫描失败       | 安全扫描失败，请重试或联系安全管理员           | 重新扫描、查看错误详情     |
| 安全等级确认超时   | 当前等待你确认安全等级总工单                   | 打开总工单、提醒处理       |
| 未选择立即采集     | 数据源已登记，可稍后继续采集表结构             | 继续采集、稍后处理         |
| 用户未填写绑定表单 | 当前等待你填写中台空间项目绑定表单             | 打开绑定表单               |
| 绑定流程失败       | 空间项目绑定失败，请重试或联系管理员           | 重新发起绑定               |

## 9. 状态查询设计

用户可以随时询问：

```text
我的数据源现在到哪一步了？
```

助手需要根据 `processId` 或用户最近任务返回：

```text
当前数据源 ds_xxx 正在等待中台空间项目绑定。

已完成：
- 数据源登记审批
- 元数据采集
- 数据地图更新

待处理：
- 请完成中台空间项目绑定

处理入口：xxx
```

如果当前卡在防火墙流程：

```text

- 元数据采集
```

```text
UAT：元数据采集中

我会分别跟踪两个环境的状态。两个环境都完成必要步骤后，会统一反馈接入准备完成。
```

## 10. 交互原型清单

第一版建议产出以下 9 张交互稿：

1. 自然语言发起数据准备任务。
2. 唤醒数据源登记表单并预填字段。
3. 表单中选择立即采集、安全扫描、同时登记 PRD/UAT。
4. 提交后展示直属领导审批和登记人复核进度。
5. 连通性失败后展示防火墙异步开通流程。
6. 登记流程办结后自动执行元数据采集和数据地图更新。
7. 安全扫描完成后等待安全等级确认总工单。
8. 等待用户填写中台空间项目绑定表单。
9. 接入准备完成，展示后续入口。

## 11. 已确认设计决策

| 问题 | 已确认结论 | 设计影响 |
| --- | --- | --- |
| 现有页面最终采用弹窗、侧边栏还是新标签页承载 | 弹窗 | 数据源登记表单和空间项目绑定表单优先通过弹窗承载 |
| 进度卡片放在助手右侧，还是作为对话卡片嵌入 | 嵌入对话卡片 | 进度信息以任务卡片形式出现在助手对话流中 |
| 审批流消息是否能提供当前处理人、待办链接和节点名称 | 可以 | 进度卡片可展示当前节点、处理人和待办入口 |
| 空间项目绑定表单是否支持预填数据源 ID、名称、负责人和所属系统 | 支持 | 绑定表单唤醒时自动预填关键字段，减少用户重复填写 |
| 后续入口链接是否可以根据数据源 ID 和空间项目绑定结果自动生成 | 可以 | 完成态可直接展示查询探索、离线开发、数据同步、安全扫描、数据脱敏入口 |
| 防火墙开通是否同步完成 | 否，对接 IT 资源管理平台，只能异步监听办结 | 需要展示防火墙流程状态，并在办结后自动重新检测连通性 |
| 登记时是否可以选择立即采集表结构和安全扫描 | 可以 | 表单中需要展示可选项；选择后进入自动采集和安全扫描分支 |
| 登记时是否支持同时登记 PRD 和 UAT 数据源 | 支持 | 进度卡片需要支持同一任务下展示 PRD/UAT 两条子流程 |
