# 数据准备专家需求文档

## 1. 需求背景

当前数据源接入已有现成的数据源登记页面、审批流程和相关后台接口，但用户仍需要理解入口、手工操作页面、跟踪审批状态，并在多个系统或模块之间串联数据源登记、元数据采集、数据地图更新、缓存更新、对外接口更新和中台空间项目绑定等操作。整体流程较长，依赖用户自行判断下一步，容易出现入口不清晰、流程遗漏、反馈不及时等问题。

为提升数据源接入效率，降低用户操作成本，计划建设“数据准备专家”。该专家不重复建设已有登记页面和审批流程，而是通过自然语言识别、页面唤醒、流程状态跟踪、现有接口编排和 MCP 服务封装，帮助用户完成从数据源登记到中台空间项目绑定的数据接入准备工作。

数据准备专家的边界不是替用户完成查询探索、离线开发、数据同步等后续使用动作，而是确保数据源完成接入准备，并具备被这些后续模块识别和使用的条件。安全扫描具备两种形态：默认作为接入完成后的后续入口；如果用户在登记时选择“立即采集表结构并进行安全扫描”，则安全扫描纳入本次数据准备流程的可选分支。

## 2. 建设目标

建设一个面向数据平台用户的数据准备专家，实现以下目标：

1. 支持用户通过自然语言触发数据源登记、注册、接入等意图。
2. 自动唤醒现有数据源登记页面，并根据用户输入预填可识别字段。
3. 复用现有登记页面的数据源信息校验能力，包括完整性、格式合法性和数据源 ID 唯一性。
4. 复用现有数据源登记审批流程，支持直属领导审批和登记人复核状态跟踪。
5. 连通性失败时，异步发起防火墙开通流程。防火墙流程对接 IT 资源管理平台，数据准备专家监听流程办结后继续推进。
6. 数据源登记流程办结后，自动触发元数据采集和数据地图更新。
7. 复用现有接口自动创建元数据采集任务，并执行表结构采集。该环节无审批流程。
8. 元数据采集完成后，复用现有接口自动更新数据地图、ES、Redis 和元数据对外接口。该环节无审批流程。
9. 支持用户在登记数据源时选择是否立即采集表结构并进行安全扫描。
10. 如果用户选择立即安全扫描，则在元数据采集完成后自动触发安全扫描，并生成安全等级确认总工单。
11. 用户完成安全等级确认后，系统提示数据源可进入数据脱敏流程。
12. 数据源登记和元数据准备完成后，引导用户通过现有表单流程绑定中台空间项目。
13. 中台空间项目绑定流程办结后，将数据源标记为“接入准备完成 / 可使用”。
14. 支持用户在登记时同时登记 PRD 和 UAT 数据源，并分别跟踪两套数据源的登记、采集、扫描和绑定状态。
15. 通过 MCP 服务包装现有接口，使智能助手能够稳定、安全、可审计地调用。
16. 向用户反馈处理进度、登记结果、数据地图查询入口和后续可选操作入口。

## 4. 用户输入

用户可通过自然语言触发数据准备专家，例如：

- 登记数据源
- 注册数据源
- 帮我新增一个 MySQL 数据源
- 我要接入一个生产环境 Oracle 数据源
- 帮我采集一个数据源的表结构

系统需要识别以下意图：

| 意图         | 示例                       | 处理方式                                         |
| ------------ | -------------------------- | ------------------------------------------------ |
| 登记数据源   | 登记数据源、注册数据源     | 唤醒现有数据源登记页面                           |
| 指定类型登记 | 新增 MySQL 数据源          | 唤醒现有登记页面并预填数据源类型                 |
| 指定环境登记 | 注册生产环境 Oracle 数据源 | 唤醒现有登记页面并预填环境和类型                 |
| 采集元数据   | 采集这个数据源的表结构     | 如数据源已存在，则进入采集流程                   |
| 登记并安全扫描 | 登记后直接采集表结构并安全扫描 | 唤醒登记页面，并预选立即采集和安全扫描选项 |
| 同时登记 PRD/UAT | 同时登记 PRD 和 UAT 数据源 | 唤醒登记页面，并进入成对数据源登记模式 |
| 查询接入进度 | 我的数据源现在到哪一步了   | 查询数据准备流程状态                             |
| 继续后续使用 | 登记完了我能做什么         | 展示查询探索、离线开发、数据同步、安全扫描等入口 |

## 5. 功能范围

### 5.1 建设边界

本需求重点建设智能助手编排能力，不重复建设已有业务系统能力。

| 能力               | 建设方式               | 说明                                             |
| ------------------ | ---------------------- | ------------------------------------------------ |
| 数据源登记页面     | 复用现有页面           | 智能助手只负责唤醒、预填和接收完成结果           |
| 数据源登记审批流程 | 复用现有流程           | 用户表单提交后进入直属领导审批、登记人复核       |
| 字段校验           | 复用现有页面和后台接口 | 包括必填、格式、唯一性、权限等校验               |
| 连通性检测         | 复用现有接口           | 可通过 MCP 工具包装后调用                        |
| 数据源登记         | 复用现有流程和接口     | 数据准备专家负责跟踪审批办结，不直接绕过流程登记 |
| 元数据采集         | 复用现有接口           | 包括创建采集任务、执行采集、查询状态；无审批流程 |
| 数据地图更新       | 复用现有接口           | 包括 ES、Redis、对外接口更新；无审批流程         |
| 安全扫描           | 复用现有接口或流程     | 用户选择后执行，扫描完成后生成安全等级确认总工单 |
| 安全等级确认总工单 | 复用现有工单流程       | 用户确认安全等级后，可进入数据脱敏流程           |
| 中台空间项目绑定   | 复用现有表单流程或接口 | 用户表单填写后进入绑定流程，数据准备专家负责引导和跟踪 |
| 流程编排与状态反馈 | 本次新增               | 由数据准备专家负责串联和反馈                     |
| MCP 服务封装       | 本次新增               | 包装现有接口，提供给智能助手调用                 |

### 5.2 专家职责边界

数据准备专家负责完成“数据源可被中台使用”的前置准备，不负责执行后续具体使用动作。

| 阶段               | 是否属于数据准备专家 | 说明                           |
| ------------------ | -------------------- | ------------------------------ |
| 数据源登记         | 是                   | 数据接入起点                   |
| 直属领导审批       | 是                   | 数据源登记流程中的审批节点     |
| 登记人复核         | 是                   | 数据源登记流程中的复核节点     |
| 连通性检测         | 是                   | 判断数据源是否可连接           |
| 防火墙流程         | 是                   | 对接 IT 资源管理平台，异步开通并监听办结 |
| 元数据采集         | 是                   | 让平台识别表结构               |
| 数据地图更新       | 是                   | 让用户可查询资产               |
| 元数据对外接口更新 | 是                   | 让高性能元数据接口可提供服务   |
| 安全扫描           | 条件属于             | 用户选择立即扫描时，属于数据准备可选分支 |
| 安全等级确认       | 条件属于             | 安全扫描后生成总工单，用户确认后可进入脱敏 |
| 绑定中台空间项目   | 是                   | 用户通过表单流程完成绑定，让数据源进入中台空间项目上下文 |
| 查询探索           | 否，作为后续推荐入口 | 属于数据使用阶段               |
| 离线开发           | 否，作为后续推荐入口 | 属于开发生产阶段               |
| 数据同步           | 否，作为后续推荐入口 | 属于链路配置阶段               |
| 安全扫描           | 条件属于，或作为后续推荐入口 | 用户登记时选择立即扫描则纳入准备流程，否则作为后续入口 |
| 数据脱敏           | 否，作为后续推荐入口 | 安全等级确认后可继续进入数据脱敏流程 |

数据准备专家的最终交付状态定义为 `READY_FOR_USE`。默认情况下，表示数据源已经完成登记、连通性检测、元数据采集、数据地图更新和中台空间项目绑定，具备被查询探索、离线开发、数据同步、安全扫描等模块使用的条件。如果用户选择立即安全扫描，则 `READY_FOR_USE` 还需要等待安全扫描完成和安全等级确认总工单办结。

### 5.3 一期范围

1. 自然语言识别数据源登记意图。
2. 唤醒现有数据源登记页面。
3. 将自然语言中识别出的环境、数据源类型等信息传递给现有页面进行预填。
4. 复用现有页面完成核心字段填写和校验。
5. 复用现有接口完成连通性检测。
6. 支持通过 MCP 服务包装现有接口，供智能助手统一调用。
7. 支持跟踪数据源登记审批流程，包括直属领导审批和登记人复核。
8. 支持登记流程办结后调用接口创建元数据采集任务。
9. 支持执行表结构采集。
10. 支持采集完成后更新数据地图、ES、Redis。
11. 支持用户选择是否立即进行安全扫描。
12. 支持用户选择后在元数据采集完成后自动触发安全扫描。
13. 支持安全扫描完成后生成安全等级确认总工单，并跟踪用户确认状态。
14. 支持安全等级确认完成后提示用户可进入数据脱敏。
15. 支持元数据准备完成后唤醒或发送中台空间项目绑定表单流程。
16. 支持跟踪中台空间项目绑定状态。
17. 支持同时登记 PRD 和 UAT 数据源。
18. 支持在绑定完成后反馈“接入准备完成”，并展示后续可选操作入口。

### 5.4 后续范围

1. 连通性失败后自动发起防火墙流程，并监听 IT 资源管理平台流程办结。
2. 支持流程状态回调和用户通知。
3. 支持更多现有接口包装为 MCP 工具。
4. 支持凭证托管、密钥引用和账号安全治理。
5. 支持自动重试、补偿任务和失败恢复。
6. 支持与数据血缘、数据安全、数据稽核等模块联动。
7. 支持专家能力沉淀为 Skill 或 Agent 工作流。

## 7. 主流程

1. 用户输入自然语言，例如“帮我登记一个 MySQL 数据源”。
2. 系统识别用户意图为“数据源登记”。
3. 系统唤醒现有数据源登记页面，并根据用户输入预填已知字段。
4. 用户在现有页面以表单形式填写数据源信息并提交。
5. 现有登记页面和后台接口进行表单校验：
   - 必填项是否完整
   - 字段格式是否合法
   - 数据源 ID 是否唯一
   - 用户是否具备登记权限
6. 表单提交成功后，进入现有数据源登记审批流程：
   - 直属领导审批
   - 登记人复核
7. 现有页面或审批流将流程编号、当前节点和待办链接回传给数据准备专家。
8. 数据准备专家关闭登记弹窗或提示用户“数据源登记流程已提交”，并持续跟踪审批流程状态。
9. 数据源登记流程办结后，系统执行连通性检测。
10. 如果连通性检测通过，系统进入自动准备阶段：
    - 如果用户选择立即采集表结构，则通过现有接口创建元数据采集任务，执行表结构采集。
    - 表结构采集完成后，通过现有接口更新数据地图、ES、Redis 和元数据对外接口。
    - 如果用户同时选择立即安全扫描，则在元数据采集完成后自动触发安全扫描。
    - 安全扫描完成后生成安全等级确认总工单。
    - 如果用户未选择立即采集表结构，则进入“待采集表结构”状态，并提示用户后续可继续采集；此时不自动更新数据地图，也不自动触发安全扫描。
11. 元数据采集、数据地图更新和安全扫描执行环节不需要用户审批或表单填写，由系统自动执行并反馈进度。
12. 如果生成安全等级确认总工单，用户需要完成安全等级确认。
13. 安全等级确认完成后，系统提示用户可以继续进入数据脱敏流程。
14. 元数据和数据地图准备完成后：
   - 检查是否需要绑定中台空间项目
   - 如需绑定，则唤醒或发送空间项目绑定表单流程给用户
   - 向用户反馈当前进入“待绑定中台空间项目”状态
15. 用户在现有空间项目绑定页面以表单形式填写绑定信息并提交。
16. 空间项目绑定流程办结后，系统接收回调或查询绑定流程状态。
17. 绑定完成后，系统将数据源状态更新为 `READY_FOR_USE`。
18. 系统向用户反馈数据源已完成接入准备，并展示查询探索、离线开发、数据同步、安全扫描、数据脱敏等后续可选入口。

如果用户选择同时登记 PRD 和 UAT 数据源：

- 数据准备专家需要在同一个 `processId` 下生成 PRD 和 UAT 两条子任务。
- 两条子任务分别跟踪登记审批、连通性检测、元数据采集、数据地图更新、安全扫描和空间项目绑定状态。
- 进度反馈需要同时展示 PRD 和 UAT 的当前状态，并明确是否存在某个环境阻塞。

如果数据源登记或连通性检测失败：

- 展示失败原因。
- 异步发起防火墙开通流程，该流程对接 IT 资源管理平台。
- 向用户反馈防火墙流程编号和当前处理状态。
- 监听防火墙流程办结消息或定时查询流程状态。
- 网络开通后自动重新检测连通性，并继续推进自动准备流程。

### 7.1 页面回调要求

现有登记页面需要向数据准备专家回传以下事件，用于驱动后续自动化流程：

| 页面事件     | 触发时机               | 回传内容                                  | 后续动作                 |
| ------------ | ---------------------- | ----------------------------------------- | ------------------------ |
| 页面打开成功 | 登记页面加载完成       | 会话 ID、页面状态                         | 展示填写中状态           |
| 用户提交成功 | 表单校验通过并提交成功 | 数据源 ID、数据源类型、环境、连接信息摘要 | 关闭弹窗，进入连通性检测 |
| 用户取消     | 用户主动关闭或取消表单 | 会话 ID、取消原因                         | 终止流程                 |
| 页面提交失败 | 页面或接口返回失败     | 错误码、错误信息                          | 展示失败原因，允许重试   |

页面提交成功后的流程衔接建议采用“回调接口为主、流程消息为辅、状态查询兜底”的机制：

1. 回调接口：现有页面提交成功后，主动通知数据准备专家进入下一步。
2. 流程消息：审批流或办结消息同步发送给用户和数据准备专家，用于用户提醒和系统状态推进。
3. 状态查询：当回调或消息丢失时，数据准备专家可根据流程编号查询当前状态。

## 10. 现有接口与 MCP 服务封装建议

说明：数据源登记、连通性检测、元数据采集、数据地图更新等能力已有现成接口，本章节不要求重新设计业务接口，重点是梳理需要包装成 MCP 工具的能力，使智能助手能够稳定、安全地调用。

### 10.1 MCP 封装原则

1. 优先复用现有业务接口，不重复建设登记、采集、地图更新能力。
2. MCP 服务只做能力包装、参数适配、鉴权透传、结果标准化和错误信息归一。
3. MCP 工具需要暴露清晰的名称、用途、输入参数、输出结构和错误码。
4. 涉及密码、Token、凭证等敏感信息时，MCP 工具只传递凭证引用，不传递明文。
5. 所有 MCP 调用需要记录审计日志，便于追踪智能助手执行过程。
6. 对长耗时任务，例如元数据采集、数据地图更新，建议提供任务创建接口和任务状态查询接口。

## 11. 页面与交互设计

### 11.1 入口

用户在智能助手中输入登记诉求后，系统唤醒现有数据源登记页面。推荐采用“统一工作台页面协议 + 弹窗或侧边栏承载现有页面”的方式：

1. 表单字段较少或用户需要保持对话上下文时，优先采用弹窗。
2. 表单字段较多、需要展示较长说明或流程状态时，优先采用侧边栏。
3. 如果现有页面无法嵌入，则允许通过新标签页打开，但必须携带 `sessionId/processId`，确保页面提交后可以回到数据准备流程。
4. 页面唤醒时应传入用户身份、会话 ID、流程 ID 和可预填字段。
5. 账号、密码、Token 等敏感信息不得通过 URL 明文传递。

### 11.2 表单交互

1. 根据自然语言识别结果自动预填字段。
2. 复用现有页面的必填字段标识和校验逻辑。
3. 复用现有页面针对不同数据源类型的差异化字段展示。
4. 提交前进行现有前端基础校验。
5. 提交后由智能助手或现有页面展示流程进度。

### 11.3 进度反馈

系统需要展示当前处理进度：

1. 页面唤醒中
2. 用户填写中
3. 表单提交中
4. 直属领导审批中
5. 登记人复核中
6. 数据源登记流程办结
7. 元数据采集中
8. 数据地图更新中
9. 空间项目绑定表单已发送
10. 等待用户填写空间项目绑定表单
11. 等待空间项目绑定流程办结
12. 接入准备完成或失败

### 11.4 完成反馈

完成后向用户展示：

- 数据源 ID
- 数据源名称
- 数据源类型
- 所属环境
- 采集表数量
- 数据地图查询入口
- 中台空间项目绑定状态
- 当前状态：接入准备完成 / READY_FOR_USE
- 后续可选操作入口：查询探索、离线开发、数据同步、安全扫描、数据脱敏

```text

你现在可以继续：
1. 查询探索：查看表结构和样例数据
2. 离线开发：基于该数据源创建开发任务
3. 数据同步：配置同步链路
4. 安全扫描：扫描敏感字段和安全风险
5. 数据脱敏：在安全等级确认后配置脱敏规则
```

如果仍等待用户绑定空间项目，反馈示例：

```text
流程编号：PROC_xxx
处理入口：xxx

完成绑定后，数据源将进入接入准备完成状态。
```

## 12. 权限与安全

1. 用户必须具备数据源登记权限。
2. 生产环境数据源登记需要更严格的权限校验。
3. 密码不允许明文展示、明文存储或在日志中输出。
4. 建议对接统一凭证管理能力，使用凭证引用代替明文密码。
5. MCP 服务调用现有接口时，优先透传用户身份和工号，确保审批流、权限校验和审计记录对应真实操作人。
6. 所有关键操作需要记录审计日志。
7. 防火墙申请、数据源登记、元数据采集等操作需要记录操作人和时间。
8. 如底层接口必须使用服务账号调用，MCP 层也必须显式传入 `operatorUserId/jobNo`，并在日志中记录真实用户与服务账号的映射关系。

## 14. 验收标准

1. 用户输入“登记数据源”后，系统能够识别意图并唤醒现有数据源登记页面。
2. 用户输入包含环境或数据源类型时，系统能够将相关字段传递给现有页面进行预填。
3. 用户在现有页面提交成功后，页面能够将登记结果回传给数据准备专家。
4. 页面提交成功后，登记弹窗能够关闭或进入下一步状态反馈。
5. 表单必填项缺失时，现有页面能够阻止提交并提示具体字段。
6. 数据源 ID 重复时，系统能够提示用户修改。
7. 数据源登记表单提交后，系统能够跟踪直属领导审批和登记人复核状态。
8. 数据源登记审批流程办结后，系统能够自动进入连通性检测和元数据准备阶段。
9. 连通性失败时，系统能够反馈失败原因，并发起防火墙申请流程或引导用户进入现有处理流程。
10. 连通性通过后，系统能够通过 MCP 工具调用现有接口创建并执行元数据采集任务。
11. 元数据采集完成后，系统能够通过 MCP 工具调用现有接口更新数据地图、ES、Redis 和元数据对外接口。
12. 元数据准备完成后，系统能够判断是否需要绑定中台空间项目。
13. 如需绑定中台空间项目，系统能够唤醒或发送空间项目绑定表单流程给用户，并展示流程编号和处理入口。
14. 用户完成空间项目绑定后，系统能够接收回调或查询绑定状态。
15. 空间项目绑定完成后，系统能够将数据源状态更新为 `READY_FOR_USE`。
16. 用户能够看到完整处理进度。
17. 用户选择立即安全扫描时，系统能够在元数据采集完成后自动触发安全扫描。
18. 安全扫描完成后，系统能够生成安全等级确认总工单，并提醒用户确认。
19. 用户完成安全等级确认后，系统能够提示可以继续进行数据脱敏。
20. 用户选择同时登记 PRD 和 UAT 数据源时，系统能够分别跟踪两套数据源的流程状态。
21. 处理完成后，用户能够看到数据源信息、数据地图查询入口和后续可选操作入口。
22. 后续可选操作入口包括查询探索、离线开发、数据同步、安全扫描、数据脱敏。
23. 任一环节失败时，系统能够给出明确错误提示和下一步建议。
24. 操作过程中的关键节点可追踪、可审计。
25. MCP 服务能够对现有接口返回结果进行标准化，便于智能助手理解和编排。

### 15.1 P0：PRD 定稿前必须确认

| 问题 | 当前回答 | 诊断结论 | 优化建议 |
| --- | --- | --- | --- |
| 现有数据源登记页面的唤醒方式是什么：弹窗、路由跳转、iframe、侧边栏，还是统一工作台页面协议？ | 推荐采用统一工作台页面协议承载现有页面 | 建议采用“统一工作台页面协议 + 弹窗或侧边栏承载现有页面”。这样既能保留用户当前对话上下文，也方便页面回调和流程状态展示 | 优先使用侧边栏或弹窗打开现有页面；如果表单较长，推荐侧边栏或新标签页；无论采用哪种形态，都需要统一传入 `sessionId/processId` |
| 现有登记页面是否支持通过 URL 参数、页面事件或上下文对象传入预填字段？ | 用户输入时可以预填写 | 已具备预填能力，满足自然语言识别后自动带入字段的要求 | 不建议把敏感信息放在 URL 参数中；环境、类型等低敏字段可通过 URL 或上下文对象传递，账号、凭证等敏感字段应使用页面上下文或凭证引用 |
| 现有登记页面提交成功后，如何通知数据准备专家继续执行下一步：页面事件、回调接口、消息通知，还是轮询查询？ | 回调接口，同时也发送消息 | 方案可行，能够支撑自动闭环和用户提醒 | 采用“回调接口为主、审批流消息为辅、状态查询兜底”的三层机制，避免单点消息丢失导致流程卡住 |
| 现有接口清单、接口文档、鉴权方式和调用限制是什么？ | 用户触发审批流，根据工号进行登录鉴权 | 已明确以用户身份和工号为核心，说明流程具有用户权限上下文 | MCP 调用应透传用户身份和工号，所有流程发起、查询、办结状态都要记录真实用户，不建议用无差别服务账号直接代办 |
| 哪些现有接口需要优先包装为 MCP 工具？ | 优先包装流程衔接和状态查询类接口 | 一期不应替代既有审批流程，而应把数据源登记表单流程、元数据自动准备、空间项目绑定表单流程串起来，并给用户清晰反馈 | 第一优先级包装数据源登记页面唤醒、登记审批流查询、审批消息查询、空间项目绑定表单唤醒、空间项目绑定流程查询；第二优先级包装元数据采集和数据地图更新；第三优先级再考虑直接登记、防火墙等动作 |
| MCP 工具调用现有接口时，使用用户身份透传还是服务账号代理？ | 用户身份 | 判断正确，适合审批流、权限审计和生产环境操作 | 采用用户身份透传；如底层接口必须服务账号调用，也应在 MCP 层绑定 `operatorUserId/jobNo`，保证审计可追溯 |
| 中台空间项目绑定流程是否已有现成入口或接口？ | 有 | 可以支撑数据准备专家完成 `READY_FOR_USE` 闭环 | 一期应复用现有入口或接口，不新建绑定能力；重点包装发起绑定、查询绑定状态、生成待办入口 |
| 空间项目绑定完成后，是否有回调通知，还是需要轮询查询流程状态？ | 有审批流消息 | 可用于驱动状态更新，但仅依赖消息存在丢失或延迟风险 | 审批流消息用于实时推进；同时保留流程状态查询工具，支持用户主动问进度和系统补偿检查 |
| `READY_FOR_USE` 状态由哪个系统维护，是否需要回写到统一数据源或数据地图？ | 统一数据源流程也会有办结通知 | 统一数据源流程可以作为重要事件源，但最终状态建议由数据准备专家聚合判断 | `READY_FOR_USE` 不建议只等同于统一数据源办结，应定义为：数据源登记审批流程办结 + 元数据采集完成 + 数据地图更新完成 + 中台空间项目绑定流程办结；如果用户选择立即安全扫描，还需要安全扫描完成和安全等级确认总工单办结 |

P0 诊断结论：当前回答已经可以支撑产品文档初稿和方案评审。数据准备专家的核心价值应定义为“串联既有流程、跟踪状态、自动推进无审批步骤、给用户清晰反馈和下一步引导”，而不是替代既有审批流程。仍需在详细设计阶段补充页面唤醒协议、回调接口字段、审批流消息格式和各 MCP 工具的真实接口映射。
