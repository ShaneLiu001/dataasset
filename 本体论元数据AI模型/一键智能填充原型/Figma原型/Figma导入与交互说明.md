# 数据资产盘点后台智能预填与确认 Figma 原型说明

## 1. 交付文件

### 原有版本

| 文件 | 用途 |
|---|---|
| `00_全流程画板_Figma.svg` | 四个步骤横向排列，用于方案评审和全流程展示 |
| `01_一键填充入口_Figma.svg` | 打开工单即展示后台预填结果和完成时间 |
| `02_填充配置_Figma.svg` | 只看待处理项，核对中低置信度和待补充字段 |
| `03_生成中_Figma.svg` | 查看生成依据，确认或修改候选值 |
| `04_生成结果与依据_Figma.svg` | 完成必填项后确认提交 |
| `generate-figma-svg.mjs` | 从同级 `Axure原型/原型四步骤母版.svg` 重新生成本目录素材 |

### 安全等级确认新增版本

以下文件为新增版本，用于展示“字段中文名/备注后台预填 + 字段安全等级 1/2/3/4 级推荐 + 用户确认”的完整流程。原有版本文件保留不覆盖。

| 文件 | 用途 |
|---|---|
| `00_全流程画板_安全等级确认_Figma.svg` | 四个步骤横向排列，展示带安全等级确认的全流程 |
| `01_一键填充入口_安全等级确认_Figma.svg` | 打开工单即展示字段安全等级推荐、需确认数量和预填结果 |
| `02_填充配置_安全等级确认_Figma.svg` | 只看待处理项，聚焦中低置信度、安全等级需确认和待补充字段 |
| `03_生成中_安全等级确认_Figma.svg` | 查看安全等级推荐依据，包括接口命中规则、列名匹配和确认建议 |
| `04_生成结果与依据_安全等级确认_Figma.svg` | 完成字段信息和安全等级确认后提交 |
| `generate-figma-security-svg.mjs` | 生成带安全等级确认的新版本素材，不覆盖原有版本 |

所有独立 SVG 已展开公共页面元素。导入 Figma 后，文字、矩形、线条和颜色均可继续编辑，不是整张位图。

## 2. 导入方法

1. 在 Figma 中新建 Design file，页面命名为 `数据资产盘点后台智能预填与确认`。
2. 若使用原有版本，将 `01` 至 `04` 四张 SVG 拖入画布；全流程评审需要时再导入 `00`。
3. 若使用安全等级确认版本，将 `01_一键填充入口_安全等级确认_Figma.svg` 至 `04_生成结果与依据_安全等级确认_Figma.svg` 拖入画布；全流程评审需要时再导入 `00_全流程画板_安全等级确认_Figma.svg`。
4. 分别选中四张独立 SVG，执行 **Frame selection**，Frame 尺寸保持 `1920 × 1080`。
5. 原有版本 Frame 依次命名为 `P01_打开工单`、`P02_核对待处理项`、`P03_查看生成依据`、`P04_确认提交`。
6. 安全等级确认版本 Frame 依次命名为 `P01_打开工单_安全等级确认`、`P02_核对待处理项_安全等级确认`、`P03_查看安全定级依据`、`P04_确认提交_安全等级确认`。
7. 字体优先使用 `PingFang SC`；缺失时使用 `Microsoft YaHei` 或系统无衬线字体。

## 4. 编辑规范

- 标签列、输入框和值区域沿用源文件固定坐标，不移动冒号位置。
- 页面主容器、编辑弹窗、配置弹窗和结果摘要不使用大圆角。
- 字段表头与数据行保持相同列宽。
- AI 标签完整显示为 `AI生成｜置信度：高/中/低`，不缩写为 `AI·高/中/低`。
- 安全等级标签完整显示为 `安全推荐｜高/中/低置信`，不与普通 AI 生成标签混用。
- 字段安全等级使用 1/2/3/4 级下拉框样式；3/4 级或中低置信度结果必须保留人工确认提示。
- 中、低置信度结果必须保留人工确认提示和生成依据入口。

## 5. 重新生成

在仓库根目录执行：

```bash
node "本体论元数据AI模型/一键智能填充原型/Figma原型/generate-figma-svg.mjs"
```

生成带安全等级确认的新版本：

```bash
node "本体论元数据AI模型/一键智能填充原型/Figma原型/generate-figma-security-svg.mjs"
```

生成后应使用 XML 校验工具检查 SVG，并在 Figma 中抽查中文字体、标签宽度、字段列、安全等级列和依据抽屉的对齐情况。
