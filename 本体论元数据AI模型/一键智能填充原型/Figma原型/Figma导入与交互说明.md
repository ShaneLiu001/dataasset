# 数据资产盘点后台智能预填与确认 Figma 原型说明

## 1. 交付文件

| 文件 | 用途 |
|---|---|
| `00_全流程画板_Figma.svg` | 四个步骤横向排列，用于方案评审和全流程展示 |
| `01_一键填充入口_Figma.svg` | 打开工单即展示后台预填结果和完成时间 |
| `02_填充配置_Figma.svg` | 只看待处理项，核对中低置信度和待补充字段 |
| `03_生成中_Figma.svg` | 查看生成依据，确认或修改候选值 |
| `04_生成结果与依据_Figma.svg` | 完成必填项后确认提交 |
| `generate-figma-svg.mjs` | 从同级 `Axure原型/原型四步骤母版.svg` 重新生成本目录素材 |

四张独立 SVG 已展开公共页面元素。导入 Figma 后，文字、矩形、线条和颜色均可继续编辑，不是整张位图。

## 2. 导入方法

1. 在 Figma 中新建 Design file，页面命名为 `数据资产盘点后台智能预填与确认`。
2. 将 `01` 至 `04` 四张 SVG 拖入画布；全流程评审需要时再导入 `00`。
3. 分别选中四张独立 SVG，执行 **Frame selection**，Frame 尺寸保持 `1920 × 1080`。
4. Frame 依次命名为 `P01_打开工单`、`P02_核对待处理项`、`P03_查看生成依据`、`P04_确认提交`。
5. 字体优先使用 `PingFang SC`；缺失时使用 `Microsoft YaHei` 或系统无衬线字体。

## 3. Prototype 连线

| 起始 Frame | 触发区域 | 动作 | 目标 Frame |
|---|---|---|---|
| `P01_打开工单` | 只看待处理项 | On click / Navigate to / Instant | `P02_核对待处理项` |
| `P02_核对待处理项` | 置信度标签 | On click / Navigate to / Instant | `P03_查看生成依据` |
| `P03_查看生成依据` | 确认该字段 | On click / Back | `P02_核对待处理项` |
| `P02_核对待处理项` | 确定按钮 | On click / Navigate to / Instant | `P04_确认提交` |
| `P04_确认提交` | 返回检查 | On click / Back | `P02_核对待处理项` |

演示 Flow starting point 设置在 `P01_打开工单`。

## 4. 编辑规范

- 标签列、输入框和值区域沿用源文件固定坐标，不移动冒号位置。
- 页面主容器、编辑弹窗、配置弹窗和结果摘要不使用大圆角。
- 字段表头与数据行保持相同列宽。
- AI 标签完整显示为 `AI生成｜置信度：高/中/低`，不缩写为 `AI·高/中/低`。
- 中、低置信度结果必须保留人工确认提示和生成依据入口。

## 5. 重新生成

在仓库根目录执行：

```bash
node "本体论元数据AI模型/一键智能填充原型/Figma原型/generate-figma-svg.mjs"
```

生成后应使用 XML 校验工具检查五张 SVG，并在 Figma 中抽查中文字体、标签宽度和各字段列对齐情况。
