# 数据资产盘点后台智能预填与确认 Axure 原型规格

## 1. 交付说明

本目录中的 `index.html` 是基于现有盘点页面制作的可交互高保真原型。智能补全在工单创建或资产分派后由后台自动执行，用户打开工单时直接确认候选结果，不再主动点击“一键智能填充”。

本机已安装 Axure RP 11。本目录已生成 `数据资产盘点一键智能填充原型.rp`，采用单页 2×2 四状态完整画板组织，并已由 Axure RP 11 重新打开验证。最新对齐规则对应 `数据资产盘点一键智能填充原型.drawio`、`原型四步骤母版.svg` 和四张独立 PNG；产品人员可按需在 Axure 中继续拆分为独立页面。

Axure 页面导入素材：

- `01_一键填充入口.png`
- `02_填充配置.png`
- `03_生成中.png`
- `04_生成结果与依据.png`

## 2. 页面与动态面板

| Axure 对象 | 类型 | 状态/页面 |
|---|---|---|
| `P01_盘点任务详情` | 页面 | 展示基本信息、盘点信息、资产表格和审批节点 |
| `DP_编辑盘点任务` | 动态面板 | `预填结果`、`仅待处理项`、`提交确认` |
| `DP_生成依据` | 动态面板 | `隐藏`、`右侧抽屉` |
| `RP_字段列表` | 中继器 | 默认、仅待处理项 |
| `Toast_操作提示` | 动态面板 | 表单校验、低置信度提示、提交提示 |

## 3. 核心元件命名

| 元件名 | 类型 | 说明 |
|---|---|---|
| `Tag_后台预填状态` | 状态标签 | 显示预填完成时间、部分待补充或预填失败 |
| `Btn_重试预填` | 次要按钮 | 仅在失败或超时场景出现，不覆盖人工修改值 |
| `Txt_表中文名` | 文本框 | AI 无法判断时保留为空 |
| `Txt_表备注` | 文本框 | 支持 AI 候选与人工编辑 |
| `Tag_AI置信度` | 标签 | 完整显示“AI生成｜置信度：高/中/低”三种样式 |
| `Btn_只看待处理项` | 文字按钮 | 过滤低置信度和待补充字段 |

## 4. 交互用例

### 用例 A：打开待办并载入预填结果

```text
点击“处理确认”
  -> 显示 DP_编辑盘点任务 / 预填结果
  -> 读取后台预填状态、候选值、置信度和证据
  -> 若存在中低置信度或待补充项，默认突出显示待处理数量
```

### 用例 B：筛选待处理项

```text
点击 Btn_只看待处理项
  -> RP_字段列表过滤 Confidence <> high 或 GenerateStatus = missing
  -> 按钮文字切换为“查看全部字段”
```

### 用例 C：查看生成依据

```text
点击字段后的 Tag_AI置信度
  -> 显示 DP_生成依据 / 右侧抽屉
  -> 写入字段、候选值、置信度、证据和确认建议
```

### 用例 D：人工修改

```text
修改 AI 候选输入框
  -> Tag_AI置信度 改为“人工修改”
  -> 设置中继器字段 IsManual = true
  -> 后台重试和人工触发重试均不覆盖该值
```

### 用例 E：确认提交

```text
点击“确定”
  -> 若表中文名或字段中文名称为空：阻止提交并定位
  -> 若存在低置信度：显示确认提示
  -> 校验通过：沿用原盘点审批流程提交
```

## 5. 中继器字段

| 字段 | 示例 | 用途 |
|---|---|---|
| `FieldName` | `cityid` | 技术字段名 |
| `FieldType` | `int(11)` | 字段类型 |
| `ChineseName` | `城市编码` | 最终显示值 |
| `Remark` | `城市的编码标识...` | 最终显示值 |
| `AiChineseName` | `城市编码` | AI 原始候选 |
| `AiRemark` | `城市的编码标识...` | AI 原始候选 |
| `Confidence` | `medium` | 高、中、低 |
| `Evidence` | `字段名拆解...` | 生成依据 |
| `GenerateStatus` | `generated` | generated、missing、failed |
| `IsManual` | `false` | 是否被人工修改 |

## 6. 视觉规范

- 主要按钮：`#356FCA`
- AI 高置信度：`#2F9B63`
- AI 中置信度：`#356FCA`
- AI 低置信度/待确认：`#D99525`
- 页面分割线：`#E3E7ED`
- 结果摘要背景：`#F2F7FF`
- 输入框高度：38 px
- 弹窗宽度：1160 px，视口不足时左右保留 24 px
- 字段列表应处理全量分页数据，不能只生成当前可见页
- 标签使用固定宽度列并右对齐，冒号统一放在标签末尾；同一区域所有冒号保持同一垂直线。
- 输入框、选择框和值区域使用统一起始线；字段表头与数据行使用相同固定列宽。
- 页面主容器、编辑弹窗、配置弹窗和结果摘要不使用大圆角；按钮等小控件可保留 4 px 轻微圆角。
- 一个步骤对应一张图，分别为打开工单查看预填结果、核对待处理项、查看生成依据、确认提交。

## 7. 演示路径

1. 从待办列表点击“处理确认”，直接打开已完成后台预填的工单。
2. 查看预填摘要、完成时间和不同置信度字段。
3. 点击“只看待处理项”定位中低置信度与待补充字段。
4. 点击任一 `AI生成｜置信度：高` 或 `AI生成｜置信度：中` 标签查看生成依据。
5. 修改或确认候选值，观察人工修改标记。
6. 补齐无法判断的必填字段后点击“确定”。
7. 在提交确认弹窗中确认进入后续审批流程。
